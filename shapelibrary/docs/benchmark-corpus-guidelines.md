# Benchmark Corpus Guidelines v0.1

The first benchmark set should prioritize signal quality over volume.

## Rules

1. Include primitive, assembly, and unknown episodes.
2. Include adversarial episodes (metaphor-heavy, contradictory, or low-observable prompts).
3. Keep expected outcomes explicit and machine-readable.
4. Keep episode payloads consistent with canonical IR shape.
5. Track changes to benchmark episodes as versioned fixtures.
