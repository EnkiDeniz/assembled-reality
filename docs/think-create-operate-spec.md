## Summary

This document defines the live workflow framing for Loegos:

`Think → Create → Operate`

This loop is now the clearest way to understand work inside an opened Box.

It does **not** replace the Box model. It sits on top of it.

- `Box` remains the container
- `Sources`, `Assembly`, and `Receipts` remain the core nouns
- `Seven` remains the conversational surface
- `Operate` remains the box-read engine

## Why This Framing Exists

The product already contains the right primitives:

- sources
- listening
- Seven
- staging
- Assembly
- Operate
- receipts

`Think → Create → Operate` makes those primitives legible as one act:

- first understand the material
- then shape something from it
- then read the box honestly and decide what happens next

## Canonical Loop

### Think

`Think` is the orientation and reading phase.

It includes:

- opening a source
- listening
- reading block by block
- asking Seven about the active document
- selecting blocks
- understanding the shape of the material before construction

### Create

`Create` is the shaping phase.

It includes:

- staging selected blocks
- moving accepted Seven output into staging
- assembling a working document
- editing blocks directly
- refining the Assembly as the active artifact in the Box

### Operate

`Operate` is the diagnosis and next-move phase.

It includes:

- running the box read
- projecting `Aim`, `Ground`, and `Bridge`
- naming `Gradient`
- naming trust floor and ceiling in `L1–L3`
- identifying convergence state
- drafting a receipt
- asking Seven to audit the read

`Operate` is not chat, not summary, and not rewrite. It is the box reading itself.

## Relationship To The Box Model

The Box model remains the underlying product structure:

- `Box` = container
- `Sources` = what came in
- `Assembly` = what is being shaped
- `Receipts` = what sealed or was drafted as proof
- `Seven` = contextual conversation
- `Operate` = box read

`Think → Create → Operate` is the user loop that moves through that structure.

## Current Surface Model

### Outside the loop

- `Boxes` chooses the container
- `Box home` orients and resumes one container

### Inside the loop

#### Think surface

- source rail
- source reading
- listening
- Seven conversation tied to the active document

#### Create surface

- staging
- Assembly
- edit/save behavior

#### Operate surface

- Operate action
- Operate result view
- Seven audit follow-up

#### Receipts surface

- latest proof
- local vs remote draft status
- GetReceipts connection state
- evidence trail

## Rules

### 1. Think should feel calmer than Create

Think is slower, quieter, and more interpretive.

### 2. Create should feel more tactile than Think

Create is where structure changes.

### 3. Operate should feel conclusive, not noisy

Operate should feel like a decisive read, not another chat panel.

### 4. Seven and Operate must stay distinct

Seven:

- talks
- interprets
- audits

Operate:

- reads the Box
- projects structure
- pressures the next move

### 5. The deeper coordinate system remains analysis, not navigation

`△ □ œ × 1–7` belongs underneath the workflow, not in front of it.

## Working Sentence

**Think in the box. Create the assembly. Operate on reality.**
