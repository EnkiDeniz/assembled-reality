## 2. Processing fluency: compression feels true

The subjective ease with which a statement is processed increases the probability that it is judged to be true. The effect operates across every domain where humans weigh information, without conscious awareness.

Three determinants: repetition, structure, perceptual features. A compressed, rhythmic, structurally clean statement is processed more easily. That ease is misattributed to the content itself. People do not think "this is easy to process." They think "this is true."

Validation: compression increases uptake because the brain prefers it. Warning: the brain cannot distinguish compression that reflects real structure from compression that merely sounds good. Settlement logic exists to add an external check where the brain's internal check is unreliable.

**Key research:** Reber and Schwarz (1999). Schwarz (2012) feelings-as-information theory. Goldman (1986) reliable-process account.

## 3. The Keats heuristic: beauty signals validity

McGlone and Tofighbakhsh (1999) presented participants with aphorisms in rhyming form ("Woes unite foes") or semantically equivalent non-rhyming form ("Woes unite enemies"). The rhyming versions were consistently judged more accurate. Comprehensibility ratings were identical.

The Keats heuristic: people assess a statement's truth based on its aesthetic qualities. The effect operates unconsciously — participants denied any awareness of rhyme influencing their judgments.

The effect extends beyond rhyme. Chiastic structure — words in the first half inverted in the second half — produces the same fluency-driven increase in perceived accuracy. This is the structure of "Move follows presence" and "Reality speaks in receipts." Not rhyming, but using inversion and compression to create the same processing advantage.

Neuroimaging suggests a candidate mechanism. fMRI studies show greater activation in the claustrum — a region associated with multimodal integration — during fluency-based judgments. If the claustrum integrates sensory streams into unified percepts, and compressed sentences activate it, then the neural processing of such sentences may resemble what "runs" means: entering as one unit, producing an orientation shift without requiring decomposition. This is inference, not established finding.

**Key research:** McGlone and Tofighbakhsh (1999, 2000). Kara-Yakoubian et al. (2022) on chiastic structure. Volz, Schooler, and von Cramon (2010) on claustrum activation.

## 4. Implementation intentions: compressed sentences change behavior

Peter Gollwitzer's research, spanning thirty years, demonstrates that formatting a plan as a compressed if-then sentence dramatically increases execution probability. The format: "If situation X arises, then I will initiate action Y." Not a goal statement. A compressed operator with a trigger condition and an action.

Meta-analysis: more than 8,000 participants across 94 independent studies. Medium-to-large effect size (d = 0.65) on goal attainment. Some domains showed large effects (d > 1.0).

The mechanism: the situation specified in the if-part becomes highly accessible once the goal is activated. A strong associative link forms between situation and action. The compressed sentence installs an automatic trigger. The person does not deliberate. The sentence has already routed the response.

"No seal, no commit" becomes implementation-intention-like when encoded as a cue-action rule: "If no seal is present, then do not commit." The compressed form carries the same structural components — a specified condition and a specified response. Once internalized, it runs without deliberation.

Vague or uncompressed plans do not produce this effect. "If I have time, I'll exercise" does not work. "If it's 7 AM on a weekday, I'll do a 15-minute workout" does work. Compression is functional, not stylistic. The density creates the associative link.

**Key research:** Gollwitzer (1993, 1999). Gollwitzer and Sheeran (2006) meta-analysis. Wieber, Thürmer, and Gollwitzer (2015).

## 5. Kolmogorov complexity: compression is optimality

The Kolmogorov complexity of an object is the length of the shortest program that produces it as output. The Minimum Description Length principle: the best model for a data set is the one that compresses it most while preserving predictive power.

Compression is not convenience. It measures how much structure exists. A data set that can be compressed contains regularities. The more it compresses, the more can be learned from it. A data set that cannot be compressed is random — no learnable structure.

A sentence that achieves maximum compression while still producing the correct behavioral output is the optimal model of the pattern it captures. The design rule "if removing one word breaks the line, you are close to the right density" is an intuitive version of this formal principle.

Kolmogorov complexity is uncomputable. No algorithm can determine the true shortest program for any given data. You can approximate and test. You cannot prove. This is the formal equivalent of fallibilism: you can always be wrong about whether you have reached the true minimum. If an operator sentence cannot fail, it cannot be tested. If it cannot be tested, you cannot know whether the compression is real.

**Key research:** Kolmogorov (1963). Solomonoff (1964). Rissanen (1978).

## 6. Prefrontal gating: the brain runs compressed rules

The prefrontal cortex governs flexible behavior through gating. Working memory selectively admits, maintains, and deploys information through gate operations — input gating (what enters), maintenance (what stays), output gating (what guides action).

When the gate is open, information flows into working memory and serves as context for action selection. When closed, irrelevant information is kept out. Mediated by cortico-striatal-thalamo-cortical loops.

The rules that operate these gates are compressed. The prefrontal cortex stores abstract rules — category-level representations that apply across multiple specific situations. Hierarchical rule use supports complex action selection, learning, planning, and decision-making.

"No seal, no commit" provides a compressed rule that the prefrontal gating system can apply across contexts. The compression matches the format the brain already uses to route behavior.

Developmental research: improvements in rule-guided behavior from childhood through adolescence correspond to maturation of gating mechanisms. The capacity to hold and apply compressed rules develops. Operator sentences are not just compatible with how the brain works — they are training material for the system that produces flexible behavior.

**Key research:** Frank, Loughry, and O'Reilly (2001). Badre and D'Esposito (2007). Unger et al. (2016). Mante, Sussillo, Shenoy, and Newsome (2013).

## 8. The derivation bridge

Operator sentences can be derived from receipt chains, not just authored by humans.

**Receipt chains as state sequences.** A complex process generates receipts: evidence that something happened. Each receipt changes the state. In the simplest limiting case, this is a Markov chain. The real object is usually a Directed Acyclic Graph — a DAG. Parallel branches converge at gates. When two independent branches converge at the same gate, the operator governing that gate has a higher assembly index than one governing a single path. The DAG gives compression more surface area, not less.

**Compression surfaces operators.** Apply Kolmogorov approximation to the receipt chain. If receipts repeatedly show "confirmed with stakeholder before proceeding," the compressed pattern is an operator: "No seal, no commit." Nobody authored it. The receipts prove it is running. If the chain shows confirmation consistently skipped, the compression surfaces an absent operator — a gap where a gate should be. The state sequence stutters. Corrections multiply. The assembly index drops.

**Derivation is stronger than authorship.** It requires no claim of insight. The receipts write the operators. The operators were assembled by the process itself and made visible through compression.

**The diagnostic:**

- Clean operators emerge: the process has structure. It is assembling.
- No clean operators emerge: the process is accumulating, not assembling. Bricks without a house.
- Operators contradict each other: competing structures. Unresolved conflict.
- Operators don't match the stated ones: the process says one thing and does another. The gap is the real diagnostic.

**The receipt schema.** Four fields: prior state (what was true before), action (the delta), operator (what rule governed it — can be empty), evidence (what changed, with seal).
