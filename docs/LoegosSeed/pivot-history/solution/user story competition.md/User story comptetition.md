# Cursor Agent   
  
ou land on Start. You tap Add source, choose paste, and drop in your markdown: it opens with *“I want to hack the code of reality”* and continues with fragments from Assembled Reality, Echo Canon, the monolith line, and the rest.  
What happens next  
1. Intake completes The file is stored as a source document in your box—same text you pasted, nothing reinterpreted yet. You are not dropped into the full compiler.  
2. Source View (calm) You see your title (from the file or first line), the full raw text, and a quiet player so you can listen without “entering a mode.” One line says something like: *This is the source as captured.* Primary action: Next: Shape seed. Secondary, quiet: Open full workspace if you already want the power tools.  
3. Shape Seed You move into a guided seed surface—not a blank advanced room. You’re writing the working object Loegos will read: Aim, what’s here, the gap, what “sealed” would mean—short prompts, one sentence each section, tied to the source you just added (you can glance back). You’re not running Operate yet; you’re authoring the position the system will analyze.  
4. Founder Shell (artifact-first) Once the seed has real text, you’re on the Founder Shell around that seed:  
    * Center: compact blocks—language first, light structure, no dashboard on every line.  
    * One next-step CTA at this stage is likely Run Operate (or “Check this seed”)—singular, explained in one line.  
    * You select a block: a small action tray appears; the Loegos read opens in the rail (or sheet on mobile): signal, why, evidence, what’s weak, override state—for that block only.  
    * Player stays available; Seven is a collapsed bubble if you want help, without owning the screen.  
5. Operate runs You run Operate. Findings attach to blocks with honest coverage (including “we didn’t evaluate everything” if that’s true). Selecting a block shows what the system sees vs your sentence—the “text lights up” moment: grounded, strained, missing evidence, etc., depending on your rules.  
6. You respond You might inspect, attest where you disagree with the model, adjust the seed, or re-run Operate. Receipt / seal stay later in the story—only when the CTA and copy say you’re ready, not because the dashboard is always visible.  
7. Graduation When you need trees, receipts, diagnostics depth, or assembly lane, you hit Open full workspace and land in Full Box with the same project, same documents, same runs—more chrome, same truth.  
  
In one breath: You pasted source → you read and heard it calmly → you shaped a seed → you entered Founder Shell around that seed → you ran Operate and used selection + Loegos read to see what the system actually claims about your language → you deepen or seal when the product offers that step, or you open Full Box when you want the whole machine.  
That’s the converged story your specs are aiming at: your file stays yours first, then Loegos earns the right to show its read, one block at a time, one next move at a time.
