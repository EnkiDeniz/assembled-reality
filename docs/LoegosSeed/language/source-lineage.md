# Source Lineage

Date: April 6, 2026
Purpose: Map every concept in the language spec back to its origin in the founding documents.

The language spec formalized what was already written. This file proves it.

## Shape types

### Language spec

Four types: Aim (△), Reality (□), Weld (œ), Seal (𒐛).

### Origin: Assembled Reality v0.7, Section 7

> Triangle governs aim.
> Square governs constraint and proof.
> Circle governs story and mediation.

The shapes were defined in the founding document as the three axes of a seven-move coordination space. The language spec adapted them: triangle became Aim, square became Reality, circle became Weld (the convergence between aim and reality), and seal (𒐛) was added as the closure type — the irreversible commitment.

### Origin: Assembled Reality v0.7, Section 12

> Set shape: Name aim (△), reality (◻), story (○). Rate each 1-7.

The Alignment Game already treated shapes as typed annotations on coordination moves. The language spec renders them as inline glyphs on text blocks.

## Signal states (color)

### Language spec

Five states: green (grounded), amber (partial), red (unsupported), neutral (uninformed), attested (human-asserted).

### Origin: Assembled Reality v0.7, Section 10

> A move becomes real when reality answers and that answer can be preserved.

The green/amber/red signal model is a formalization of the document's core distinction: receipt-backed (green) vs candidate (amber/red) vs uninformed (neutral). The document says "everything without return is candidate, not fact." The language spec renders that distinction as text color.

### Origin: Assembled Reality v0.7, Section 5

> Everything without a receipt is candidate, not fact. Not false — unverified.

This is the exact definition of the amber signal: "not false — unverified." The language spec adds color. The concept was already there.

## Signal irreversibility

### Language spec

Once a signal moves from neutral to any informed state, it cannot return to neutral.

### Origin: Assembled Reality v0.7, Section 12

> align → seal → stand on the seal → declare next △ → new gap → align again.
> The 0 you return to is not the same 0 you started from.

The document says you can return to the starting position but standing on a sealed platform that didn't exist before. The language spec translates this into rendering: a block that has been informed can change signal but can never go back to uninformed. Knowledge accumulates; it doesn't forget.

## Trust levels

### Language spec

L1 (unverified), L2 (partial provenance), L3 (verified, multi-source).

### Origin: Assembled Reality v0.7, Section 10

> Gradient as assembly depth: level 1 is simple closure, low cost. Level 7 is multi-domain, multi-agent convergence — difficult to fake.

The 1-7 gradient was already a trust/depth model. The language spec simplifies to three levels for visual rendering (L1/L2/L3) while preserving the gradient concept in the formal model.

## Weight as trust × depth

### Language spec

Weight combines trust (L1-L3) and depth (D1-D4) into a visual property: thin/light → normal → bold/solid.

### Origin: Assembled Reality v0.7, Section 9

> Courage without receipts is theater. Courage with receipts is traction.
> A sealed receipt is earned grip.

"Earned grip" IS visual weight. The document describes text that gains substance through evidence and sealing. The language spec renders that substance as font weight and spacing density.

## Seal as irreversible closure

### Language spec

Seal (𒐛) is a closure type. Once sealed, a block's weight never decreases.

### Origin: Assembled Reality v0.7, Section 12

> The seal is a foundation, not a finish line.

The document already defines seal as permanent foundation. The language spec renders it as visual permanence — bold, solid, immutable-feeling text.

## The gap between declaration and return

### Language spec

The product makes meaning visible by rendering the gap between what the user wrote and what the system sees.

### Origin: Assembled Reality v0.7, Section 2

> The universal failure mode of coordinating intelligences is coherence without contact.
> The core product problem is misalignment between thought-shape and reality-shape.

The entire product concept — making the gap between intention and reality visible — is stated in the second section of the founding document. The language spec makes that gap visible through color: amber text is coherence without contact. Green text is coherence WITH contact.

## Type mismatch detection

### Language spec

If a Reality-typed block reads as pure intention, the system flags it as a type error.

### Origin: Assembled Reality v0.7, Section 14

> Mirror: △↔△ — Is this aim actually mine?
> Crossing: △→◻ — Does my aim survive contact with the evidence?
> Crossing: ◻→△ — Is the evidence asking me to change direction?

The pre-seal audit matrix already checked for type mismatches: aim misidentified as evidence, evidence that should change the aim, story that replaced evidence. The language spec automates this as a type-checking system.

## Canon as posture, not source

### Language spec (via Inference Authority Rules)

The Echo Canon and Monolith Cosmology shape how Seven communicates but never appear as cited evidence.

### Origin: Assembled Reality v0.7, throughout

The document itself embodies this rule: it states principles like "friction is testimony" and "wipe the story, keep the receipts" as behavioral dispositions, not as evidence for specific claims. The language spec formalizes what the document practices: canon governs tone, not findings.

## The triangulation principle

### Language spec

Meaning lives between three vertices: human language, Loegos language, and reality.

### Origin: Assembled Reality v0.7, Section 2

> The human authors. The AI assists. Reality closes.
> Neither the human nor the AI is the ground. The ground is return.

Three vertices: human, AI, reality. The language spec names them as the triangulation that produces meaning. The concept was already in Section 2 of the founding document.

## Summary

Every concept in the language spec traces back to the Assembled Reality document. The language spec is not an invention. It is a compilation — the founding document compiled into rendering instructions.

The product that compiles human language into coordination code was itself compiled from source text that already contained the coordination code. The spec was always there. We just hadn't run Operate on it yet.
