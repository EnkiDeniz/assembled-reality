# Learning So Far: Working Echo, Test Drive II

Date: April 11, 2026  
Status: Learning note

## Summary

This is the first Test Drive II result I would call a real product win.

The latest clean run says:

- benchmark valid: `yes`
- headline valid: `yes`
- official winner under the current acceptance rules: `loegos_sighted`

The important part is not the badge alone.

The important part is that `loegos_sighted` is now real product signal, not fake signal:

- `loegos_sighted`: `81.25`
- `loegos_blindfolded`: `66.25`
- `plain_chat`: `58.75`
- `structured_chat`: `55`
- `schema_board`: `85`

So the visible echo now matters in a real way.

But the run also says something equally important:

**we are not yet best-in-class at explicit evidence mapping.**

That is the remaining hill.

## What We Now Know Is True

### 0. Aim belongs back at the center

The latest run sharpened the echo.
The next read sharpens again if we put aim back where it belongs.

Without aim:

- there is no meaningful miss
- there is no meaningful correction
- there is no meaningful receipt

So the product should now be understood as an aimed reality loop:

**Aim -> Echo -> Split -> Witness/Test -> Return -> Receipt -> Re-aim -> Canon**

Where:

- aim gives direction
- the echo shows the current balance between aim and reality
- the deciding split and missing witness give the steering correction
- return is reality answering
- receipt is proof from reality
- re-aim is honest course correction
- canon is what survives law

### 1. The existence question is basically answered

The question used to be:

> does showing the echo matter at all?

Now the answer is basically yes.

The visible working echo:

- beats blindfolded Lœgos overall
- beats plain chat overall
- meets or exceeds structured chat under the current acceptance rules
- materially improves product-native cases like:
  - `contradictory_return_journey`
  - `working_echo_correction`

### 2. The law loop is not the weak part

The system is not weak on:

- contradiction handling
- counterfeit resistance
- premature-move avoidance
- return-aware steering
- deciding the next split

That means the chassis is doing its job.

### 3. The remaining weakness is thinner receipts

The main miss is not judgment. It is evidence explicitness.

We often have the right read, but still with thinner receipts than the best board.

That shows up most clearly in:

- evidence carry-forward
- support vs weaken separation
- honest-fog cases where the right move is still restraint

### 4. `schema_board` is still a real teacher

Even though `loegos_sighted` is the official winner under the current rules, `schema_board` still has the highest raw mean score at `85`.

That means the control is still teaching us something valuable:

- harder evidence handles
- more brutal `supports / weakens / missing`
- clearer receipt shape in fog

The biggest visible gains came from cases where the surfaced echo helped the operator steer, not just read:

- `contradictory_return_journey`
  - `loegos_sighted`: `90`
  - `loegos_blindfolded`: `65`
- `working_echo_correction`
  - `loegos_sighted`: `85`
  - `loegos_blindfolded`: `55`

That tells us the current surfaced echo is now doing real work in:

- return-aware steering
- visible correction
- next-move placement

## Where We Are Still Weak

The remaining miss is narrower than "the system is worse."

### 1. Honest fog

`no_move_yet` is still the clearest weak corner:

- `loegos_sighted`: `65`
- `loegos_blindfolded`: `65`
- `schema_board`: `80`

That says the current echo is still too summary-shaped when the right answer is:

- do not conclude yet
- name what weakens the easy story
- name the one missing witness that would decide it

### 2. Evidence explicitness

The mean evidence metrics tell the real story:

- `loegos_sighted` mean `Evidence`: `6.25`
- `schema_board` mean `Evidence`: `10`
- `loegos_sighted` mean `Evidence Split`: `7.5`
- `schema_board` mean `Evidence Split`: `10`

So the next improvement is not generic intelligence.

It is:

- harder evidence handles
- less gist
- stronger provenance
- cleaner `supports / weakens / missing`

### 3. Cost

We are also carrying more weight:

- `loegos_sighted` median tokens: `2673`
- `structured_chat`: `1289`
- `plain_chat`: `1214.5`

That is acceptable if the steering gain is worth it.
It is not itself a reason to retreat.
But it means the echo has to earn its weight.

## What Changed My Mind

### 1. Externalization is necessary, but not sufficient

I used to think that once the echo was visible, Lœgos would probably prove itself.

Now I think:

**visibility is necessary, but the visible surface still has to carry harder receipts than chat or a generic board.**

### 2. The benchmark fairness problem is no longer the main story

Earlier, unfair setup was the dominant caveat.

Now the benchmark is clean enough that the more important truth is:

**the current visible echo is real and useful, but the next gains have to come from stronger evidence mapping, not from arguing about the benchmark.**

### 3. The real competition is not "any board"

The strongest competition now looks like:

- excellent structured chat
- and a strong external evidence board

That means the real question is not:

> can we beat plain conversation?

It is:

> can we build an echo that steers better than the best structured alternative while preserving the law loop?

## The 911 Read

The right optimization lens now is not generic goodness.

It is:

**what makes this machine unmistakably itself under load?**

### North stars

- steering feel
- braking
- road contact
- reality loop
- proof travel

### Guardrails, not goals

- lowest token count
- fastest first answer
- prettiest prose
- quietest interface
- softest friction

If a change does not make the operator steer better, trust the return more, or carry proof more cleanly, it is probably secondary right now.

## What The Next Wave Should Optimize For

### 1. Make `whatWouldDecideIt` even stronger

This is the steering rack.

Especially in `no_move_yet`, it should point to:

- one deciding split
- one missing witness
- one artifact that would actually separate the reads

### 2. Make evidence harder, not prettier

The echo should show:

- what the current aim seems to be
- what supports the current read
- what weakens the popular story
- what evidence is still missing

And it should do that with stronger handles and provenance.

### 3. Make return shifts unmistakable

Returns should visibly change the read:

- what changed
- what prior read weakened
- how the current aim changed
- what next move changed

### 4. Add lightweight correction

The operator should be able to place the car better, not just watch it.

### 5. Add bounded future help in the right roles

- GetReceipts as proof status and portability
- Shape Library as bounded advisory for deciding split and receipt condition

Neither should become another authority mouth.

## One Line

We are no longer trying to prove that the echo matters.

We are trying to make the echo carry harder receipts than anything else on the road while keeping aim visibly revisable under reality.
