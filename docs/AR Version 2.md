# Loegos Version 2

**Status:** Working next-stage direction document  
**Date:** April 2026

**Companion roadmap:** `docs/AR Version 2 Build Plan.md`

## One-Line Thesis

Loegos Version 2 extends the `1.0` Box workbench into a fuller box lifecycle:

`open box → gather sources → shape assembly → return from reality → let Seven read the pattern`

## What Version 2 Builds On

Version `1.0` already proves the product is real.

The shipped workbench already supports:

- auth
- box-based workspace language
- source intake
- listening
- per-document Seven conversation
- staging
- assembly creation
- receipt drafting
- optional GetReceipts connection

Version 2 should not reopen the basic naming or structure questions. Those are settled:

- the top-level object is `Box`
- the active artifact is `Assembly`
- the proof layer is `Receipts`
- Seven is contextual and conversational first
- `△ □ ○ × 1–7` is an analysis model, not the primary navigation model

## The V2 Direction

Version 2 should make the box feel more complete over time, not more confusing.

The right direction is:

1. make box identity and navigation first-class
2. complete the return-from-reality loop
3. deepen Seven from document conversation into box reading
4. strengthen composition so assemblies feel authored
5. strengthen receipts as the durable proof layer

## Primary Product Hierarchy

The V2 hierarchy remains:

### 1. Box

The native top-level object and home for one unit of real work.

### 2. Sources

The raw material inside the box.

### 3. Assembly

The active built artifact the user is shaping.

### 4. Receipts

The proof and history layer for what was assembled and what happened next.

### 5. Seven

The conversation and diagnostic layer that reads what is in the box.

## What Version 2 Must Improve

### 1. Box management

The product now has Boxes, but the experience needs to feel more intentional:

- better box creation flows
- clearer switching between boxes
- stronger box naming
- cleaner box home / launcher behavior

### 2. The return-from-reality moment

This is still the biggest gap.

The product is strong at helping a user make the thing. It is weaker at helping them:

- take the assembly into the world
- come back later
- record what happened
- compare outcome to intention

Version 2 must make return feel first-class.

### 3. Seven as box reader

Today, Seven is primarily a document conversation surface.

That is correct for `1.0`. But Version 2 should start to let Seven read across the box:

- what is in the box
- what is missing
- what is overrepresented
- what pattern is beginning to emerge

This should happen gradually and only when the box has enough material to justify the read.

### 4. Composition quality

Assemblies are real, but they still lean too hard toward ordered selection.

Version 2 should make assemblies feel more shaped:

- stronger connective tissue
- clearer structure
- better synthesis tools
- more intentional templates and composition patterns

### 5. Receipts as payoff

Receipts already exist, but Version 2 should make them feel more central:

- easier to review
- easier to trust
- easier to export or publish
- more legible as proof rather than secondary metadata

## What Should Stay Stable

These parts of the product are already right and should stay stable:

### The block model

Blocks remain the atomic unit of assembly.

### Listening

Listening remains load-bearing and should continue to feel native, not ornamental.

### Staging

Staging remains the key intermediate memory surface.

### Human control

AI may propose. The user still selects, stages, and assembles.

## V2 Relationship To The Deeper Doctrine

The Lœgos coordinate system remains strategically important.

But Version 2 should introduce it through use, not ideology.

The correct order is:

1. make the box useful
2. let Seven read the box
3. surface deeper analysis only when it helps

Not:

1. teach symbols first
2. make users navigate by doctrine

The doctrine should deepen the product, not obscure it.

## Success Criteria For Version 2

Version 2 succeeds if:

- Boxes feel first-class and understandable
- Seven becomes more useful because it can read more than one document at a time
- users can return from reality and capture what happened
- assemblies feel more composed
- receipts feel more valuable over time
- the product stays calmer as it grows more capable

## Working Summary

Version 1 made the Box workbench real.

Version 2 should make the Box lifecycle real.
