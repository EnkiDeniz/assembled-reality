This repository is in an intentional product pivot.

The purpose has stayed stable, but the product center has changed. The current direction is no longer best described as a workspace made of phases and panels. It is now better described as a system for:

- capturing witness material quickly
- listening and marking what matters
- creating a visible working echo outside the chat
- converting that emerging echo into lawful Lœgos structure
- choosing bounded moves
- testing them in reality
- preserving receipts of what actually held

That working echo should not read like a decorative summary. The first steering surface should make three things plainly visible:

- what seems real
- what conflicts
- what would decide it

## North Stars

These are the things we should optimize like a driver's car, not a luxury sedan:

- `Steering feel`
  - does the working echo help place the next move better?
- `Braking`
  - do contradiction and premature-closure resistance stay elite?
- `Road contact`
  - can the operator see what supports, what weakens, what is missing, and what would decide it?
- `Reality loop`
  - do returns visibly change the read?
- `Proof travel`
  - can the read carry proof and trust cleanly across time and handoff?

## Guardrails, Not Goals

These matter only when they get bad enough to block use:

- lowest token count
- fastest first answer
- prettiest prose
- quietest interface
- softest friction

They are not the reason this machine exists.

## Current Benchmark Truth

The latest clean Test Drive II run now says:

- benchmark valid: `yes`
- headline valid: `yes`
- official winner under current acceptance rules: `loegos_sighted`
- strongest remaining control pressure: `schema_board` still has the highest raw mean score (`85` vs `81.25`)
- strongest remaining product miss: honest fog and thinner receipts, especially in `no_move_yet`

So the next lap is not about proving the visible echo matters at all. It does.

The next lap is about making the visible echo carry harder receipts and sharper deciding splits than excellent structured chat or a generic external board.

One promising future legibility mode is a reverse trace that renders the whole interaction as a Lœgos signal field and gives stronger signal to what survived more law and reality contact.

The first practical pre-test for that idea is Drive Tape v0: a replay instrument built from current Test Drive II artifacts that helps us optimize for signal survival rather than prose quality alone.

The latest founder and engineer reverse-trace reads now add a stronger thesis:

- we were already running the loop in natural language
- the product is catching up to the language
- the next phase should make that survival structure more legible before adding heavier machinery

## Language First

Before we build any new mechanism, ask:

> how does Lœgos itself already solve this?

The steering rule now is:

- the language owns meaning
- the machine owns legibility, persistence, and binding
- reality owns proof

If the lawful artifact already knows something, surface that before adding a new heuristic layer to infer it again.

## Testing Lanes

The next testing architecture now has four lanes:

1. `Law`
   - constitutional guarantees, no fake echo, no premature canon
2. `Language Fidelity`
   - is the machine surfacing what Lœgos already knows, or inventing a parallel layer?
3. `Product Short Loop`
   - current Test Drive II, focused on deciding split, missing witness, honest fog, aim, and re-aim
4. `Product Long Loop`
   - delayed Test Drive III after the short loop is clearly ours

`state -> move -> test -> receipt -> update -> seal / flag / stop -> return`

`rate -> tighten weakest axis -> rerate -> seal / flag / stop`

## Important rule

Do not treat the current shell as the product definition.

Treat the existing backend, intake, listening, language, Operate, override, and receipt substrate as reusable assembly parts for the next product shape.

Also do not treat Seven's chat response as the whole product. The thing we are trying to make distinct is the visible working echo beside the chat and the lawful movement from that echo into accepted truth.

And if the product does not visibly show it, the benchmark should not be allowed to see it either.
