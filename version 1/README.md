## The current backbone

Three layers now define the system:

### Product

`state -> move -> test -> receipt -> update -> seal / flag / stop -> return`

### Language

`declare -> capture -> observe -> weld -> move -> test -> receipt -> seal`

### Agents

`rate -> tighten weakest axis -> rerate -> seal / flag / stop`

These layers should not be collapsed into one another.

## Platform stance

### Mobile

Mobile is primarily for:

- fast source addition
- reading
- listening
- following text while audio plays
- marking blocks that matter to the aim

### Desktop

Desktop is primarily for:

- witness inspection
- explicit compile to Lœgos
- compare
- structural editing
- move/test judgment
- receipt review

## How to use this folder

If you are:

- redefining the product
- building the next shell
- deciding what Seven should do
- choosing what the user actually does on mobile vs desktop

start here, not in the legacy workspace docs.

## Relationship to the rest of the repo

- `version 1/` is the current strategic center for the pivot
- `docs/` holds the prior runtime canon and supporting specs
- `docs/LoegosSeed/` preserves the seed and development trail
- `language/` remains important for language/rendering reference, but it is not the only product center anymore

## Important rule

Do not treat the older shell shape as the product.

Treat the existing backend, intake, listening, language, Operate, override, and receipt substrate as reusable parts for the new product definition.
