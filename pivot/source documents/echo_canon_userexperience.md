## What This Is

This document is the human doctrine of Lœgos.

It is not the database schema.
It is not the API contract.
It is not the engine spec.

It is the language layer that tells the team what the system believes, what the user is being invited into, and what kinds of sentences should survive when the product is simplified down to its most durable truths.

Use this document for:

- product philosophy
- product language and copy
- landing, About, and orientation surfaces
- internal alignment when the product starts to sprawl
- deciding which actions feel native to Lœgos and which feel like generic workspace software

Do **not** use this document alone to define persistence, state transitions, or compute logic. That belongs in [Echo Protocol](./echo-protocol.md).

## How To Use This Canon

### 1. For product copy

Prefer sentences that sound like this canon rather than generic productivity language.

Good:

- "Seal receipt"
- "Draft proof"
- "Keep the gap visible"
- "Reality speaks in receipts"

Avoid:

- "Mark complete"
- "Task status"
- "Finalize item"
- "Generate summary"

### 2. For interaction design

The canon favors:

- one obvious move
- reversible steps before irreversible seals
- compact naming
- evidence before narrative
- quiet authority, not gamified urgency

### 3. For team alignment

When product decisions feel ambiguous, ask:

- Does this increase presence or noise?
- Does this clarify the aim?
- Does this strengthen evidence?
- Does this keep the trace intact?
- Does it deserve a seal?
- Does it leave room for grace?

### 4. For architecture

The canon is a source of constraints, not a substitute for formal model design.

Translate canon into system rules through [Echo Protocol](./echo-protocol.md).

## Relationship To Current Product Terms

The canon does not replace the current runtime language. It deepens it.

Current product nouns remain:

- Root
- Seed
- Receipt
- Assembly Index
- Box

The canon tells us how those nouns should feel:

- Root should feel like declared aim
- Seed should feel like living composition
- Receipt should feel like witnessed proof
- Seal should feel like quiet closure
- Assembly Index should feel like the record that survives

That translation layer is formalized in [Echo Protocol](./echo-protocol.md).
